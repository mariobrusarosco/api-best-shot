# Almanac Database Schema

## Status And Authority

This document contains only accepted database schema for the new Almanac application. Broad ideas
remain in `docs/backend-contract-notes.md`; open and proposed decisions remain in slice documents.

Drizzle schema declarations and generated migrations are the executable implementation of this
document. A mismatch between them is an error to resolve before continuing.

## PostgreSQL Boundary

All Almanac-owned tables use the `almanac` PostgreSQL schema.

## Explicit Non-Persistence Decision

The Almanac Contents section metadata is backend configuration owned by the Contents service. It
does not have a PostgreSQL table, Drizzle schema declaration, repository, or migration. The service
uses the Editions service's public count behavior only to derive the Teams index page number.

## `almanac.world_cup_editions`

Owner: Editions domain.

The host flag field is implemented by migration `0004_almanac_edition_host_flag_asset_key`.

| Column | Type | Null | Rules |
| --- | --- | --- | --- |
| `id` | `uuid` | No | Primary key; generated by PostgreSQL |
| `source_key` | `text` | No | Unique stable source identity |
| `year` | `smallint` | No | Unique; between 1930 and 2100 |
| `name` | `text` | No | Edition display name |
| `host_display_name` | `text` | No | Current accepted host display value |
| `host_flag_asset_key` | `text` | Yes | Provider-neutral host-country flag key used by the Editions index |
| `created_at` | `timestamptz` | No | Defaults to `now()` |
| `updated_at` | `timestamptz` | No | Defaults to `now()` |

The Editions index orders rows by `year DESC`.

The API derives these values instead of storing them:

```text
path         = /editions/{year}
pageNumber   = first edition page + ordered array index
host.flagUrl = ASSET_BASE_URL + host_flag_asset_key
```

## `almanac.world_cup_edition_visual_identities`

Owner: Editions domain.

This table stores the optional visual identity for one World Cup edition.

| Column | Type | Null | Rules |
| --- | --- | --- | --- |
| `edition_id` | `uuid` | No | Primary key; foreign key to `world_cup_editions.id`; cascades on edition deletion |
| `logo_asset_key` | `text` | Yes | Provider-neutral edition-logo asset key |
| `trophy_asset_key` | `text` | Yes | Provider-neutral edition-trophy asset key |
| `accent_color` | `text` | No | Edition accent color |
| `accent_text_color` | `text` | No | Text color intended for the accent color |
| `spine_color` | `text` | No | Edition spine color |
| `created_at` | `timestamptz` | No | Defaults to `now()` |
| `updated_at` | `timestamptz` | No | Defaults to `now()` |

Relationship:

```text
world_cup_editions 1 -> 0..1 world_cup_edition_visual_identities
```

An edition may exist without a visual-identity row. When the row exists, all three color fields are
required while both asset keys remain nullable.

The API derives these values instead of storing them:

```text
logoUrl   = ASSET_BASE_URL + logo_asset_key
trophyUrl = ASSET_BASE_URL + trophy_asset_key
```

Page background and surface colors are not part of the accepted schema. Visual-identity values are
engineer-owned data; the schema and migration do not define or backfill them.

## `almanac.national_teams`

Owner: Teams domain.

Implemented by migration `0003_almanac_national_teams`.

This table represents reusable national-team identity, such as Brazil or Argentina. It does not
represent an edition-specific player roster.

| Column | Type | Null | Rules |
| --- | --- | --- | --- |
| `id` | `uuid` | No | Primary key; generated by PostgreSQL |
| `code` | `varchar(3)` | No | Unique national-team code, such as `BRA` |
| `display_name` | `text` | No | Public team name, such as `Brazil` |
| `flag_asset_key` | `text` | Yes | Provider-neutral public flag key |
| `created_at` | `timestamptz` | No | Defaults to `now()` |
| `updated_at` | `timestamptz` | No | Defaults to `now()` |

The Teams index orders rows by `display_name ASC`.

The API derives these values instead of storing them:

```text
path       = /teams/{lowercase code}
pageNumber = first team page + ordered array index
flagUrl    = ASSET_BASE_URL + flag_asset_key
```

## Current Relationships

The Editions domain owns a one-to-zero-or-one relationship from `world_cup_editions` to
`world_cup_edition_visual_identities`.

The Participations domain connects `world_cup_editions` and `national_teams` through
`world_cup_edition_teams`:

```text
world_cup_editions 1 -> many world_cup_edition_teams
national_teams     1 -> many world_cup_edition_teams
```

These same-product foreign keys follow ADR 0003. Tournament-specific squads and appearance
aggregation require future accepted slices and are not represented in the current schema.

## `almanac.players`

Owner: Players domain.

This table stores reusable player identity. It does not represent membership in a specific World
Cup squad.

| Column | Type | Null | Rules |
| --- | --- | --- | --- |
| `id` | `uuid` | No | Primary key; generated by PostgreSQL |
| `source_key` | `text` | No | Unique stable source identity |
| `given_name` | `text` | Yes | Player given name when the validated source provides one |
| `family_name` | `text` | No | Player family or known display surname |
| `display_name` | `text` | No | Derived seed value used for display |
| `birth_date` | `date` | Yes | Player birth date when known |
| `wikipedia_url` | `text` | Yes | Source reference URL when known |
| `created_at` | `timestamptz` | No | Defaults to `now()` |
| `updated_at` | `timestamptz` | No | Defaults to `now()` |

The seed derives `display_name` as:

```text
given_name + " " + family_name, when given_name exists
family_name, otherwise
```

## `almanac.world_cup_edition_teams`

Owner: Participations domain.

Implemented by migration `0007_almanac_participations`.

Each row represents one national team's participation in one World Cup edition. The table stores
validated edition-specific finish and tournament totals; it does not represent reusable team
identity or a player roster.

| Column | Type | Null | Rules |
| --- | --- | --- | --- |
| `id` | `uuid` | No | Primary key; generated by PostgreSQL |
| `source_key` | `text` | No | Unique stable source identity |
| `edition_id` | `uuid` | No | Foreign key to `world_cup_editions.id`; deletion restricted |
| `team_id` | `uuid` | No | Foreign key to `national_teams.id`; deletion restricted |
| `final_position` | `smallint` | No | Validated source value; unique within one edition |
| `official_final_position` | `smallint` | Yes | Official source value when provided |
| `final_position_source` | `text` | No | Validated source description |
| `final_stage` | `text` | No | Last tournament stage reached by the team |
| `matches_played` | `smallint` | No | Validated tournament total |
| `wins` | `smallint` | No | Validated tournament total |
| `draws` | `smallint` | No | Validated tournament total |
| `losses` | `smallint` | No | Validated tournament total |
| `goals_for` | `smallint` | No | Validated tournament total |
| `goals_against` | `smallint` | No | Validated tournament total |
| `points` | `smallint` | Yes | Validated tournament total when provided |
| `created_at` | `timestamptz` | No | Defaults to `now()` |
| `updated_at` | `timestamptz` | No | Defaults to `now()` |

The pair `(edition_id, team_id)` is unique. Champion and title-year calculations derive from rows
whose `final_position` is `1`; no aggregate champion table is stored.

Value-level `CHECK` constraints are deferred. They can be introduced later through a forward
migration after the application requires those database-enforced rules.
